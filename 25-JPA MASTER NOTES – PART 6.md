# 39. Transactions in JPA

A transaction is a unit of work that is executed completely or not executed at all.

---

## Why Transactions?

Suppose:

```text
Save Employee
Save Address
Save Parking
```

If Address saves successfully but Parking fails:

```text
Employee → Saved
Address → Saved
Parking → Failed
```

Database becomes inconsistent.

---

## Solution

Use transactions.

```text
Either ALL succeed

OR

ALL rollback.
```

---

## Transaction Flow

```text
Transaction Starts
       ↓
Save Employee
       ↓
Save Address
       ↓
Save Parking
       ↓
Commit
```

If error occurs:

```text
Transaction Starts
       ↓
Save Employee
       ↓
Save Address
       ↓
Parking Fails
       ↓
Rollback
       ↓
Nothing Saved
```

---

# @Transactional

Used to make methods transactional.

Example:

```java
@Transactional
public void saveEmployee() {

    repo.save(emp);

}
```

---

## Internal Flow

```text
Method Called
      ↓
Spring Opens Transaction
      ↓
Business Logic Executes
      ↓
Commit
```

If Exception:

```text
Spring Opens Transaction
      ↓
Business Logic Executes
      ↓
Exception Occurs
      ↓
Rollback
```

---

# 40. Commit vs Rollback

## Commit

Meaning:

```text
Make changes permanent.
```

Example:

```sql
COMMIT;
```

---

## Rollback

Meaning:

```text
Undo all changes.
```

Example:

```sql
ROLLBACK;
```

---

## Comparison

| Feature | Commit | Rollback |
|----------|---------|-----------|
| Saves Data | Yes | No |
| Reverses Changes | No | Yes |
| Transaction Ends | Yes | Yes |
| Database Updated | Yes | No |

# 41. SQL Generation by Hibernate

Hibernate converts Java objects into SQL.

---

Example:

Java:

```java
repo.save(emp);
```

Hibernate generates:

```sql
INSERT INTO emp_details (...)
VALUES (...);
```

---

Example:

Java:

```java
repo.findAll();
```

Hibernate generates:

```sql
SELECT *
FROM emp_details;
```

---

Example:

Java:

```java
repo.delete(emp);
```

Hibernate generates:

```sql
DELETE
FROM emp_details
WHERE emp_id=?;
```

---

# Internal Flow

```text
Java Object
      ↓
Hibernate
      ↓
SQL
      ↓
JDBC
      ↓
Database
```

# 42. Sequence Generator

Used to generate primary key values automatically.

Your Example:

```java
@SequenceGenerator(
        name="addr_id",
        sequenceName="addr_id",
        allocationSize=1,
        initialValue=1)

@GeneratedValue(
        strategy=GenerationType.SEQUENCE,
        generator="addr_id")
```

---

# Why Sequence?

Without sequence:

```java
id=1;
id=2;
id=3;
```

Developer manually assigns IDs.

Error-prone.

---

With Sequence:

```text
Database automatically generates IDs.
```

---

# Internal Flow

```text
Need ID
     ↓
Hibernate asks Sequence
     ↓
Sequence returns value
     ↓
Entity assigned ID
     ↓
INSERT executes
```

---

# Example

Database Sequence:

```text
1
2
3
4
5
```

Employee Saves:

```text
Employee A → 1

Employee B → 2

Employee C → 3
```

---

# allocationSize

Example:

```java
allocationSize=10
```

Meaning:

Hibernate fetches:

```text
1–10
```

in one DB call.

---

Next:

```text
11–20
```

---

## Advantage

```text
Fewer database calls.
```

---

## Disadvantage

Unused IDs may be skipped.

---

# initialValue

Example:

```java
initialValue=100
```

Sequence begins from:

```text
100
```

---

Example:

```text
100
101
102
103
```

# 43. Generation Strategies

JPA supports four strategies.

---

## AUTO

```java
@GeneratedValue(
    strategy=GenerationType.AUTO)
```

Meaning:

```text
Provider chooses strategy.
```

---

## IDENTITY

```java
@GeneratedValue(
    strategy=GenerationType.IDENTITY)
```

Meaning:

```text
Database auto-increment.
```

Example:

```sql
AUTO_INCREMENT
```

---

## SEQUENCE

```java
@GeneratedValue(
    strategy=GenerationType.SEQUENCE)
```

Meaning:

```text
Uses database sequence.
```

---

## TABLE

```java
@GeneratedValue(
    strategy=GenerationType.TABLE)
```

Meaning:

```text
Stores IDs in separate table.
```

---

# Comparison

| Strategy | Uses |
|-----------|-------|
| AUTO | Provider decides |
| IDENTITY | Auto Increment |
| SEQUENCE | Database Sequence |
| TABLE | Separate ID Table |

Java:



---

Java:

```java
repo.deleteById(1L);
```

```text
        ↓

        ↓

        ↓
```

# JPA Exam One-Line Revision

```text
@Entity → Makes class persistent.

@Table → Maps entity to table.

@Id → Primary key.

@GeneratedValue → Generates IDs.

@OneToOne → One entity to one entity.

@OneToMany → One parent to many children.

@ManyToOne → Many children to one parent.

Cascade → Propagate operations.

FetchType → Decide loading time.

JpaRepository → CRUD abstraction.

EntityManager → Core JPA API.

Hibernate → JPA implementation.

Persistence Context → First-level cache.

Dirty Checking → Auto update detection.

@Transactional → Atomic operations.

SequenceGenerator → Automatic sequence IDs.
```
